## 目标

生成或审校 `PROJECT.md`，把项目已有的主宪章与专项宪章配置成所有 AI 编程工具的统一入口。

`PROJECT.md` 不是新的权威规则源，而是导航入口、读取顺序、任务路由和工具接入说明。强约束必须继续收口到 `.specify/memory/constitution.md` 及其登记的专项宪章中。

## 前提条件

- `.specify/memory/constitution.md` 必须存在。
- 推荐先运行 `/tac-governance-constitution` 完成主宪章与可用专项宪章配置。
- 若主宪章登记了专项宪章但文件尚不存在，本 Skill 不编造专项正文，只在 `PROJECT.md` 中标记为“待生成/待确认”。

## 执行模式

默认执行 **configure** 模式。

| 模式 | 触发语义 | 行为 |
| ---- | -------- | ---- |
| `configure` | “生成 PROJECT.md”“配置项目入口” | 读取宪章体系，生成 `PROJECT.md`，可按需更新工具入口文件 |
| `audit` | “检查 PROJECT.md”“审计项目入口” | 只检查 `PROJECT.md` 与宪章体系是否一致，不覆盖文件 |
| `refresh` | “刷新 PROJECT.md”“按现有宪章重建入口” | 重新生成 `PROJECT.md`；覆盖前必须列出变更影响 |

若用户未指定模式，按 `configure` 执行。

## 执行步骤

### 1. 读取宪章体系

必须先读取 `.specify/memory/constitution.md`，再扫描 `.specify/memory/*-constitution.md`。

提取以下信息：

| 信息 | 来源 | 用途 |
| ---- | ---- | ---- |
| 项目名称 | 主宪章标题或仓库目录名 | `PROJECT.md` 标题 |
| 主宪章路径 | `.specify/memory/constitution.md` | 最高优先级入口 |
| 专项宪章清单 | 主宪章“专项规范清单”与实际文件扫描 | 生成宪章索引 |
| 已生成专项宪章 | `.specify/memory/*-constitution.md` 实际存在文件 | 任务读取路由 |
| 待生成专项宪章 | 主宪章登记但文件不存在 | 风险提示 |
| 技术栈/包名空间/评审流程 | 主宪章“技术与合规边界”“研发流程与评审要求” | 项目摘要 |

一致性规则：

- 主宪章登记的专项宪章和实际文件不一致时，不能静默修正；必须在 `PROJECT.md` 中列出“待处理项”。
- 实际存在但主宪章未登记的专项宪章不能被当作强约束；必须标记为“存在但未登记”。
- `PROJECT.md` 不得复制宪章大段正文，只能摘要和链接，避免形成第二规则源。

### 2. 生成 PROJECT.md

`PROJECT.md` 必须包含以下结构：

```markdown
# PROJECT.md

本文件是仓库对所有 AI 编程工具的统一入口。它只负责导航、读取顺序和任务路由，不独立定义高于宪章体系的强约束。

## 规则优先级

1. `.specify/memory/constitution.md`
2. 主宪章登记且文件存在的专项宪章
3. `PROJECT.md`
4. README、doc、设计说明、操作手册等一般文档

当 `PROJECT.md` 与主宪章或专项宪章冲突时，以宪章为准，并应修正 `PROJECT.md`。

## 宪章索引

| 类型 | 文件 | 状态 | 适用范围 |
| ---- | ---- | ---- | -------- |
| 主宪章 | `.specify/memory/constitution.md` | 已生成 | 全项目非协商规则、优先级、流程入口 |
| 输入质量专项宪章 | `.specify/memory/input-quality-constitution.md` | 已生成/待生成/未登记 | AI 工具接收功能开发与 Bug 修复请求时的输入完整性闸门、追问模板、验收标准字段质量 |
| 工程规范专项宪章 | `.specify/memory/android-system-design-constitution.md` | 已生成/待生成/未登记 | 架构、分层、模块、依赖、资源、系统设计 |
| 编码规范专项宪章 | `.specify/memory/kotlin-coding-constitution.md` | 已生成/待生成/未登记 | Kotlin 源码、测试代码、Gradle Kotlin DSL、格式与命名 |

## 默认读取顺序

任何 AI 工具开始工作前，必须按以下顺序读取：

1. `.specify/memory/constitution.md`
2. 与当前任务命中的专项宪章
3. `PROJECT.md`
4. 与任务直接相关的 README、doc、spec、plan、tasks 或代码上下文

## 任务读取路由

| 任务类型 | 必读宪章 |
| -------- | -------- |
| 接收功能开发、行为变更、Bug 修复请求（任何 AI 工具入口） | 主宪章 + 输入质量专项宪章 |
| 需求、用户场景、成功标准、验收标准 | 主宪章 + 输入质量专项宪章 + 功能点分析专项宪章（若存在） |
| 架构、模块、依赖、资源、平台能力、系统设计 | 主宪章 + 工程规范专项宪章 |
| Kotlin/源码/测试/Gradle 脚本变更 | 主宪章 + 编码规范专项宪章 |
| 提交、送审、质量闸门、交付说明 | 主宪章 + 研发流程专项宪章（若存在） |
| 文档新增、迁移、归档、索引 | 主宪章 + 文档治理专项宪章（若存在） |

## AI 工具接入规则

1. 工具专属入口文件只允许作为跳转文件，指向 `PROJECT.md`。
2. 工具专属 hooks、快捷键或本地偏好可以保留在工具配置中，但不得定义与宪章冲突的项目规则。
3. 新增 AI 工具时，先复用 `PROJECT.md`，不得复制一份分叉规则。
4. 若工具无法读取 `PROJECT.md`，必须在该工具入口文件中保留最小摘要和到本文件的路径。

## 待处理项

- 主宪章已登记但文件不存在的专项宪章。
- 文件存在但未登记到主宪章的专项宪章。
- `PROJECT.md` 中引用路径不存在的文档。

## 维护规则

- 调整宪章体系后必须同步刷新或审计 `PROJECT.md`。
- `PROJECT.md` 只能做摘要和导航，不承载详细编码、架构、流程或业务规则。
- 修改规则优先级、宪章路径或任务读取路由时，必须检查主宪章是否需要同步修订。
```

生成时必须根据实际扫描结果调整“宪章索引”和“待处理项”，不能照抄示例状态。

### 3. 更新工具入口文件

扫描项目根目录和常见工具位置，识别：

- `CLAUDE.md`
- `AGENTS.md`
- `GEMINI.md`
- `.cursorrules`
- `.cursor/rules/*`
- `.github/copilot-instructions.md`

处理规则：

- 若文件不存在，不强制创建，除非用户明确要求为某个工具创建入口。
- 若文件存在且只承载项目说明，应替换为最小跳转内容。
- 若文件存在且包含 hooks、工具私有配置或用户自定义规则，不得覆盖；只提示用户人工合并。
- 不自动把工具入口文件加入 `.gitignore`。是否忽略由项目协作策略决定。

推荐最小跳转内容：

```markdown
请先阅读项目根目录的 `PROJECT.md`。它是本仓库对所有 AI 编程工具的统一入口；主宪章和专项宪章位于 `.specify/memory/`。
```

若工具入口文件位于子目录，链接路径必须按相对路径修正。

### 4. Audit 检查

`audit` 模式必须检查：

- `PROJECT.md` 是否存在。
- `PROJECT.md` 是否声明自己不是权威规则源。
- 规则优先级是否为：主宪章 > 专项宪章 > PROJECT.md > 一般文档。
- `PROJECT.md` 引用的宪章文件是否存在。
- 主宪章登记的专项宪章是否出现在 `PROJECT.md` 宪章索引中。
- 实际存在的 `*-constitution.md` 是否被 `PROJECT.md` 索引。
- 工具入口文件是否只做跳转，或是否存在与宪章冲突的项目规则。
- 是否存在 `{{`、`}}`、`TODO`、`TBD` 等未完成占位。

### 5. Refresh 写入

`refresh` 模式会重新生成 `PROJECT.md`。覆盖前必须：

1. 读取旧 `PROJECT.md`。
2. 对比将要更新的章节。
3. 保留用户在“项目备注”“本地工具提示”等明确非规则区块中的内容。
4. 告知会覆盖的路径。

若无法可靠区分用户自定义内容和生成内容，停止覆盖并输出建议合并方案。

### 6. 验证

完成后必须验证：

- `PROJECT.md` 存在，且引用 `.specify/memory/constitution.md`。
- `PROJECT.md` 中所有宪章链接路径存在，或在“待处理项”中明确列出。
- `PROJECT.md` 没有 `{{`、`}}` 残留。
- 若更新了工具入口文件，入口文件链接到 `PROJECT.md` 的相对路径正确。
- 未覆盖包含工具私有配置或用户自定义规则的入口文件。

### 7. 交付说明

完成后向用户说明：

- 生成或审计的 `PROJECT.md` 路径。
- 已索引的宪章文件。
- 待生成、未登记或路径异常的宪章文件。
- 已更新、跳过或需人工合并的工具入口文件。
- 下一步建议：若存在待生成专项宪章，先补齐对应宪章；若宪章体系已完整，可开始使用 Spec Kit 流程。

## 设计原则

- **入口不是规则源**：`PROJECT.md` 只做导航和路由，不复制宪章正文。
- **宪章驱动索引**：先读主宪章和实际专项文件，再生成入口。
- **工具中立**：不偏向 Claude、Codex、Gemini、Cursor 或 Copilot；工具入口文件只做最小跳转。
- **默认不破坏**：不覆盖包含私有配置或用户自定义内容的工具入口文件。
- **可审计**：所有缺失、未登记、路径异常和人工合并项都必须显式列出。
